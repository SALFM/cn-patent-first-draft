# cn-patent-first-draft

这是一个面向中国发明专利和实用新型专利初稿生成的 skill。它根据技术交底、附图信息、内部材料和法规/审查规则摘要，生成适合提交给专利代理师、律师或企业法务继续加工的专利初稿，包括待确认事项、术语统一表、权利要求书、说明书、摘要、附图说明与附图标记、自检报告和建议补充信息。

## 这个项目是做什么的

`cn-patent-first-draft` 用于把研发人员、项目团队或企业内部已经掌握的技术材料整理成中国发明或实用新型专利初稿。它不是检索系统，也不是授权判断系统，而是一个围绕中国专利撰写习惯和审查规则进行结构化成稿的辅助工具。

核心用途包括：

- 将技术交底整理为中国发明或实用新型专利初稿。
- 帮助判断方案更适合发明、实用新型、发明和实用新型同日申请，或存在外观设计/不授权客体风险。
- 按中国专利撰写习惯生成权利要求书、说明书、摘要、附图说明和附图标记。
- 对 AI、算法、软件、大数据类方案进行技术性与数据合规风险提示。
- 对内部材料进行名称、发明人、代理信息、摘要、审阅状态等一致性检查。
- 输出自检报告和待补充事项，方便专利代理师、律师或法务继续处理。

本项目生成的是“技术性初稿”，不是最终法律意见，不承诺授权，不替代专利代理师、律师或企业法务判断。

## 适用场景

适合使用本 skill 的情况包括：

- 用户已有技术交底，希望整理成专利初稿。
- 机械结构、系统、方法、控制逻辑、算法、软件、数据处理等技术方案需要初步成稿。
- 需要把零散研发材料转成专利撰写结构。
- 需要统一术语，梳理技术问题、技术方案、技术效果。
- 需要检查发明人、专利名称、摘要、代理信息等内部材料一致性。
- 需要先判断方案更接近发明、实用新型、同日申请，还是存在外观设计或不授权客体边界风险。

## 不适用场景和限制

本 skill 有明确边界：

- 不生成或承诺最终授权结论。
- 不编造现有技术文献、对比文件、实验数据、附图、申请人、发明人、身份证号、代理机构、优先权、申请号等信息。
- 信息缺失时使用 `[待补充]`，并在待确认事项或建议补充信息中列明。
- 对不授权客体、违法或高风险内容只做风险提示，不把风险内容包装成普通可授权方案。
- 外观设计、疾病诊断治疗方法、纯智力活动规则、纯商业规则等不适合作为普通发明/实用新型初稿处理。
- 实用新型必须围绕产品形状、构造或其结合，不能把方法、流程、算法、材料配方直接写成实用新型核心保护对象。

## 快速开始

1. 阅读或复制 `templates/technical_disclosure_form.md`，填写技术交底。
2. 准备已有材料，例如技术方案、附图清单、附图标记、实施例、现有技术、实验数据、内部审批材料等。
3. 在支持 skill 的环境中启用或调用 `cn-patent-first-draft`。
4. 输入技术交底，并明确目标，例如：

```text
请使用 cn-patent-first-draft skill，根据以下技术交底生成中国专利初稿。
请先判断更适合发明、实用新型还是同日申请。
缺失信息请使用 [待补充]，不要编造附图、实验数据、现有技术文献或主体信息。
```

5. 根据输出中的“待确认事项”和“建议用户补充的信息”继续补充材料。
6. 将初稿交由专利代理师、律师或企业法务进一步加工确认。

## 输入材料说明

建议用户尽量提供以下信息：

- 专利名称。
- 拟申请类型。
- 技术领域。
- 应用场景。
- 现有技术及缺陷。
- 要解决的技术问题。
- 核心技术方案。
- 必要技术特征。
- 可选技术特征。
- 产品结构、方法步骤、系统模块、数据流、算法模型。
- 实施例。
- 工作原理。
- 有益效果。
- 附图和附图标记。
- 申请人、发明人、代理机构、优先权、分案、涉密、保密审查等程序信息。
- AI/算法/软件/大数据方案的数据来源、数据合法性、模型流程、部署环境、输出技术结果等。

不确定的信息可以填 `[待补充]`，不要为了凑完整而编造。

## 输出内容说明

skill 通常输出以下内容：

- 一、待确认事项：列出专利类型、公开、优先权、分案、同日双报、涉密、主体信息、敏感数据、实验数据、附图等需要确认的内容。
- 二、术语统一表：统一专利名称、技术对象、部件/步骤/模块名称、附图标记和内部材料基准值。
- 三、权利要求书：用阿拉伯数字连续编号，形成独立权利要求和从属权利要求草稿。
- 四、说明书：按专利名称、技术领域、背景技术、发明内容/实用新型内容、附图说明、具体实施方式组织文本。
- 五、摘要：生成不超过 300 字的连续段落，概括名称、技术领域、技术问题、技术方案要点和主要用途。
- 六、附图说明与附图标记：仅基于真实附图整理图号、图名、图示内容和标记表；无真实附图时标注 `[待补充]`。
- 七、自检报告：按风险项、影响、建议修改或待确认的方式检查初稿质量和合规风险。
- 八、建议用户补充的信息：给出下一轮迭代最需要补充的材料清单。

```text
请使用 cn-patent-first-draft skill，基于以下技术交底生成中国发明/实用新型专利初稿。
```

## 质量和合规原则

核心原则如下：

- 不编造。
- 缺失信息显式标记。
- 技术问题、技术方案、技术效果闭环。
- 权利要求以说明书为依据。
- 说明书支持权利要求。
- 摘要不超过 300 字。
- 有附图才写附图说明，不虚构附图。
- AI/算法/软件方案必须体现技术问题、技术手段、技术效果。
- 实用新型必须有形状、构造或其结合，且通常需要结构附图。
- 内部材料一致性需要单独检查。

## 维护说明

后续维护建议：

- 更新法规或审查指南后，应先更新 `references/` 中原始资料。
- 再更新 `reference_summaries/` 中对应摘要。
- 最后同步检查 `SKILL.md`、`LEGAL_BASIS.md`、模板和 README。
- 修改业务规则时，应尽量保留可追溯依据。
- 不要直接删除历史规则，除非确认已经被新规则替代。
- 修改模板时，同步检查示例输出和自检清单。
- 涉及专利法规、审查指南或内部检查口径变化时，建议由专利代理师、律师或法务复核。

## 免责声明

本项目仅用于辅助生成专利申请文件初稿。

输出内容不构成法律意见、授权意见或代理意见。

最终申请文件应由专利代理师、律师、企业法务或相关专业人员审查确认。

用户应自行保证技术交底、实验数据、附图、主体信息和程序信息的真实性、完整性和合法性。
