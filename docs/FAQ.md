# 常见问题

## 这个 skill 能直接提交专利申请吗？

不能。它生成的是专利技术草案，用于交给专利代理师、律师、企业法务或相关专业人员继续加工。最终提交文本、正式申请字段和程序事项需要人工审查确认。

## 没有附图能不能生成实用新型？

可以。本 skill 不要求用户先提供正式附图；没有正式附图时，应输出建议附图方案，说明每张图要表达什么、图中应包含哪些部件/连接关系/动作方向/标记。不得把建议图写成已经存在的正式附图，也不得使用“如图 1 所示”作为未确认事实。

## 没有实验数据能不能写有益效果？

可以写由技术特征合理推出的技术效果，但不能写成已经由实验验证的结论。依赖实验才能成立的效果，必须提供测试条件、测试方法、对照对象和真实数据；没有数据时应标记 `[待补充]`。

## AI/算法方案怎么判断是否具备技术性？

重点看是否形成技术问题、技术手段、技术效果闭环。输入数据应具有具体技术领域中的技术含义，处理流程应体现技术手段，输出结果应是技术结果，而不只是商业、金融、营销、管理或主观分类结果。

## 可以让 skill 编造现有技术或对比文件吗？

不可以。skill 不得编造现有技术文献、对比文件、公开日期、公开方式或检索结论。缺少最接近现有技术时，应写 `[待补充]` 并提示用户补充。

## `[待补充]` 是什么意思？

`[待补充]` 表示当前没有足够技术事实依据，不能安全生成确定内容。它用于保护草案不编造、不误导，也方便用户和代理师快速定位下一步要补充的信息。

## 为什么输出里会有风险提示？

风险提示用于标出可能影响授权基础、保护范围、说明书支持、数据合规或后续正式申请处理的事项。主体/程序信息默认不进入技术草案；只有用户明确要求正式申请字段时才单独处理。风险提示不是最终法律结论，而是提醒后续人工复核。

## 发明和实用新型同日申请是什么意思？

同日申请通常指同一申请人就同样发明创造在同日分别提交发明专利申请和实用新型专利申请。需要注意，同样发明创造只能授予一项专利权；两件申请应按要求分别说明已申请另一件专利，后续发明授权时可能需要放弃已授权且未终止的实用新型。

## 这个项目能替代专利代理师吗？

不能。它只能辅助整理技术交底、生成闭环迭代草案和提示风险。最终申请策略、权利要求保护范围、法律风险和提交文本，应由专利代理师、律师、企业法务或相关专业人员审查确认。

## 方法、流程或算法能写成实用新型吗？

通常不能。实用新型保护产品的形状、构造或者其结合。方法步骤、流程、算法、控制策略、材料配方或用途不能作为实用新型的核心保护对象；这类方案通常应考虑发明方向。

## 外观设计可以用这个 skill 生成吗？

不适合作为主要用途。本 skill 主要面向发明和实用新型技术方案初稿。若创新点主要是产品形状、图案、色彩及其组合，并以美感或视觉识别为重点，应提示更接近外观设计，而不是套用发明/实用新型初稿结构。

## 申请人、发明人、代理机构这些信息必须给吗？

默认不需要。本 skill 主要写技术草案，申请人、发明人、代理机构、优先权、分案、同日双报等主体/程序信息默认不进入技术草案，也不会因为缺失触发 `NEEDS_USER_FACTS`。只有用户明确要求正式申请字段或内部材料核验时，才按用户提供的真实信息处理，不得推断或补写。

## 能不能只生成一次初稿，不做闭环？

不作为本 skill 的默认能力。本 skill 默认且必须执行结果状态驱动闭环：生成 `v0` 草案后至少进行一次审查、路由、回归检查和迭代状态决策。只有状态为 `REVISE` 才继续修复；其他停止状态输出当前最佳草案。
