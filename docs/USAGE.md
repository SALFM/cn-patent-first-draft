# 使用说明

## 推荐调用方式

可以这样要求模型使用本 skill：

> 请使用 cn-patent-first-draft skill，按照结果状态驱动的闭环模式，根据以下技术交底自动生成中国发明/实用新型专利技术草案。请先生成 v0 草案，然后进行审查和状态判断。只有当状态为 REVISE 且问题可基于现有材料修复时，才继续下一轮。状态为 PASS、NEEDS_USER_FACTS、RISK_ACCEPTED 或 STOP_LIMIT 时，请停止自我迭代并输出当前最佳草案、评分卡、迭代状态决策记录、附图绘制建议和剩余风险。附图请输出建议图方案，说明每张图要包含什么；不要为了凑轮次重复改写，也不要编造现有技术、实验数据、产品结构、参数或已存在附图。

然后粘贴技术交底。

## Pipeline Controller 调用方式

如果希望每个阶段由独立 Codex 对话执行，可以这样要求：

> 请使用 cn-patent-first-draft skill 的 Pipeline Controller 模式。请把任务拆成 Intake、Invention Mining、Feature Matrix、Claim Architecture、Specification / Drawing Plan / Initial Draft、Review / Routing、Revision / Regression Decision、Final Packaging 等阶段。每个阶段由独立 Codex 完成本阶段产物、阶段评分卡和阶段交接包。父级 controller 读取 handoff 和 scorecard 后，根据结果状态决定是否自动开启下一阶段新对话。只有 controller 可以开启下一对话；阶段 Codex 不得自行开启。只有状态为 REVISE 且 ticket 可基于现有材料修复时才进入修复阶段；PASS、NEEDS_USER_FACTS、RISK_ACCEPTED 或 STOP_LIMIT 时停止或进入最终打包。不要为了推进阶段编造现有技术、实验数据、产品结构、参数、正式附图或主体/程序信息；主体/程序信息默认不进入技术草案。

Pipeline 模式适合信息量较大、需要多阶段审查、希望保留每阶段评分和交接记录的任务。未显式启用 Pipeline 时，skill 仍在当前对话执行完整闭环迭代，不存在非迭代普通模式。

## 最小输入也可以开始

用户不需要一次性准备完整申请材料。至少建议提供：

- 技术名称或主题；
- 所属技术领域和应用场景；
- 现有方案及其缺陷；
- 要解决的技术问题；
- 核心结构、步骤、模块、数据流或控制逻辑；
- 预期技术效果；
- 是否已有附图、草图或希望建议哪些图；
- 是否涉及公开、AI、个人信息、医疗、遗传资源或其他会影响技术方案边界的风险。

缺失事项会进入 `[待补充]` 和最终“剩余风险与需用户确认事项”。

## 指定迭代强度

可在请求中明确：

- “快速强度：最多 1 个修复轮次”；
- “标准强度：最多 3 个修复轮次”；
- “深度强度：最多 5 个修复轮次”；
- “重点检查权利要求保护范围和说明书支持”；
- “重点输出附图绘制建议，正式图由我绘制”。

迭代强度只设置最大安全轮次，不表示必须做满。实际修复轮次由结果状态决定：只有 REVISE 才继续；PASS、NEEDS_USER_FACTS、RISK_ACCEPTED 或 STOP_LIMIT 均停止自我迭代。若连续两轮提升小于 0.3 分、没有可执行 routing ticket 或缺失关键信息导致无法继续提升，会输出当前可达最高质量版本和停止原因。

## 附图使用方式

skill 会输出建议附图，不要求先提供正式附图。每张建议图应包含建议图号、图名、图示内容、图中应包含的部件/模块/步骤/关系、建议标记以及对应的权利要求和说明书内容。

在正式附图未绘制并确认前，说明书不得把建议图写成已经存在的“图 1”，不得使用“如图1所示”作为确定事实。

申请人、发明人、代理机构、优先权等主体/程序信息默认不进入本技术草案，也不会因缺失触发 NEEDS_USER_FACTS；只有用户明确要求正式申请包时才处理这些字段。

## 现有技术处理

如果用户没有提供现有技术，skill 可以做“现有技术压力测试假设”，但不能写成真实检索结论。

如果用户提供现有技术，应做 element-by-element 对比，区分已知特征、区别特征和待确认项。

## 输出

默认且唯一的最终输出格式为新版闭环结果：

`# 闭环专利草案生成结果`

不得使用旧版单轮格式绕过 v0、审查、路由、回归检查和迭代状态决策。`templates/patent_draft_output.md` 仅作为历史兼容参考保留，不作为调用方式。

Pipeline 模式还会使用：

- `templates/pipeline_state.md`
- `templates/phase_scorecard.md`
- `templates/phase_handoff.md`
