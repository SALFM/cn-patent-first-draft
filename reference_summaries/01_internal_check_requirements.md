# 内部材料一致性检查规则

来源文件：`references/检查要求.docx`

本摘要仅提取来源文件中出现的内部材料及检查要求，用于后续写入 `self_checklist.md`。来源文件未提供的标准值，后续 skill 只能保留为 `[待补充]`，不得自行推断或补写。

## 涉及的内部材料类型

1. 发明专利-申请知识产权发明人排序表
2. 发明专利-知识产权申请承诺书
3. 发明专利-国防科技大学知识产权申请审批表
4. 发明专利-专利应用前景证明
5. 发明专利-专利代理委托书
6. 发明专利-说明书

## 内部材料一致性检查表

| 检查对象 | 检查字段 | 检查标准 | 常见错误 | 风险等级 | 后续 skill 中的处理方式 |
|---|---|---|---|---|---|
| 发明专利-申请知识产权发明人排序表 | 发明人 | 发明人姓名、顺序应与用户确认的最终发明人信息及其他内部材料一致；未提供最终信息时标记为 `[待补充]`。 | 发明人漏写、错写、顺序变化；与承诺书或审批表不一致。 | 高 | 提取发明人列表和顺序，与承诺书、审批表中的发明人逐项比对；不一致时列出差异并要求确认。 |
| 发明专利-申请知识产权发明人排序表 | 身份证号 | 身份证号应保持完整原样，特别是前导 0 不得丢失；未提供原始身份证号时标记为 `[待补充]`。 | Excel 或表格处理导致前导 0 被吞；身份证号位数异常；数字格式被科学计数法或自动格式化。 | 高 | 一律按文本处理身份证号，不转数字、不去除前导 0；检查位数和原文显示，发现疑似吞 0 时高优先级提示。 |
| 发明专利-知识产权申请承诺书 | 专利名称 | 专利名称不得擅自更改，应与用户确认的最终专利名称及其他内部材料一致；未提供最终名称时标记为 `[待补充]`。 | 名称被改写、简称化、错别字；与审批表、应用前景证明、委托书或说明书名称不一致。 | 高 | 建立统一的专利名称基准值，跨材料比对；没有基准值时保留 `[待补充]` 并提示需确认。 |
| 发明专利-知识产权申请承诺书 | 发明人 | 发明人信息应与发明人排序表、审批表及用户确认信息一致；未提供最终信息时标记为 `[待补充]`。 | 发明人漏写、增删、排序变化；同名人员书写不一致。 | 高 | 与发明人排序表、审批表交叉校验；差异按姓名和顺序分别列出。 |
| 发明专利-国防科技大学知识产权申请审批表 | 专利名称 | 名称不得擅自更改，应与承诺书、应用前景证明、委托书、说明书等材料一致；未提供最终名称时标记为 `[待补充]`。 | 审批表名称与其他材料不一致；名称存在错别字、遗漏限定词或版本差异。 | 高 | 将审批表名称纳入专利名称一致性校验；与所有出现专利名称的材料逐项比对。 |
| 发明专利-国防科技大学知识产权申请审批表 | 发明人 | 发明人应与发明人排序表、承诺书及用户确认信息一致；未提供最终信息时标记为 `[待补充]`。 | 发明人漏写、错写、排序不一致。 | 高 | 提取审批表发明人字段，与排序表、承诺书进行姓名和顺序一致性检查。 |
| 发明专利-国防科技大学知识产权申请审批表 | 拟选代理人 | 拟选代理人不得擅自更改，应与用户确认的代理信息一致；未提供代理信息时标记为 `[待补充]`。 | 拟选代理人被替换、姓名或机构信息错误；与委托相关材料不一致。 | 高 | 提取拟选代理人字段，与用户提供的代理信息及委托书相关信息比对；缺失时提示 `[待补充]`。 |
| 发明专利-国防科技大学知识产权申请审批表 | 摘要 | 摘要内容应与用户确认的摘要或其他材料中的摘要表述一致；未提供摘要基准时标记为 `[待补充]`。 | 摘要被改写、删减、版本不一致；与其他材料中的摘要不一致。 | 高 | 将摘要作为跨材料一致性字段处理；如多份材料出现摘要，逐字或语义核对版本差异，并提示用户确认最终版。 |
| 发明专利-专利应用前景证明 | 专利名称 | 名称应与其他内部材料中的专利名称一致；未提供最终名称时标记为 `[待补充]`。 | 名称与审批表、承诺书、委托书或说明书不一致；使用旧名称。 | 高 | 纳入专利名称一致性总表，统一比对并输出差异。 |
| 发明专利-专利应用前景证明 | 应用前景 | 应用前景内容应存在且完整；来源文件未给出具体判定标准时，缺失标准标记为 `[待补充]`。 | 应用前景为空、明显不完整、段落缺失或只保留占位文字。 | 中 | 检查应用前景字段是否为空、是否仍含占位符；具体内容是否充分由用户补充或确认。 |
| 发明专利-专利代理委托书 | 公司名 | 公司名应与用户确认的代理/受托公司名称一致；未提供标准公司名时标记为 `[待补充]`。 | 公司名错写、简称与全称混用、与代理信息不一致。 | 高 | 提取公司名字段，与用户提供的标准公司名或代理信息比对；没有标准值时不推断，保留 `[待补充]`。 |
| 发明专利-专利代理委托书 | 专利名 | 专利名应与其他内部材料中的专利名称一致；未提供最终名称时标记为 `[待补充]`。 | 委托书专利名使用旧版本、漏字、错字或与审批表/说明书不一致。 | 高 | 将委托书专利名归入专利名称一致性检查，逐项比对。 |
| 发明专利-说明书 | 说明书名称 | 说明书名称应与其他内部材料中的专利名称一致；未提供最终名称时标记为 `[待补充]`。 | 说明书标题与申请材料名称不一致；标题使用旧名称或错别字。 | 高 | 提取说明书标题/名称，与专利名称基准值及各内部材料名称比对。 |
| 发明专利-说明书 | 是否审阅 | 说明书应完成审阅；来源文件未提供审阅人、审阅日期或审阅标准时，相应信息标记为 `[待补充]`。 | 未审阅、审阅状态不明、缺少审阅记录。 | 中 | 检查是否存在明确审阅状态或审阅记录；无记录时提示 `[待补充]`，不得默认已审阅。 |

## 应写入 self_checklist.md 的规则

1. 仅检查来源文件列出的 6 类内部材料：发明专利-申请知识产权发明人排序表、发明专利-知识产权申请承诺书、发明专利-国防科技大学知识产权申请审批表、发明专利-专利应用前景证明、发明专利-专利代理委托书、发明专利-说明书。
2. 用户未提供标准值的信息，后续 skill 只能写 `[待补充]`，不得自行编造发明人、身份证号、专利名称、代理人、摘要、应用前景、公司名或审阅结论。
3. 身份证号必须按文本处理，前导 0 不得丢失；发现身份证号位数异常、显示为数字格式、科学计数法或疑似吞 0 时，应列为高风险问题。
4. 发明人必须在发明人排序表、知识产权申请承诺书、国防科技大学知识产权申请审批表之间保持一致；同时检查姓名、人数和排序。
5. 专利名称必须在知识产权申请承诺书、国防科技大学知识产权申请审批表、专利应用前景证明、专利代理委托书、说明书之间保持一致；不得使用旧名称、简称或改写名称。
6. 摘要必须以用户确认的最终摘要为准；若多份材料出现摘要，必须保持一致，未提供最终摘要时写 `[待补充]`。
7. 代理信息必须保持一致，重点检查国防科技大学知识产权申请审批表中的拟选代理人，以及专利代理委托书中的公司名；未提供标准代理信息时写 `[待补充]`。
8. 专利应用前景证明必须检查专利名称和应用前景内容；应用前景缺失、不完整或仍为占位内容时，应提示补充。
9. 说明书必须检查名称一致性和是否审阅；没有明确审阅状态或审阅记录时，不得默认已审阅，应写 `[待补充]`。
