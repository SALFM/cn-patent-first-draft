# 说明书模板摘要

来源文件：`references/专利说明书撰写要求.docx`

本摘要将说明书模板中的 `T1-T13` 槽位转化为专利初稿生成骨架。该模板适合生成中文专利说明书的基础版本，但生成时必须根据专利类型、技术主题、是否有附图、是否为方法类方案进行适配，不能机械套用原文。

## 1. 说明书结构

### 1.1 专利名称

模板首行使用：

`一种(T1)【】`

生成时应将 `T1` 展开为完整专利名称。若用户输入已包含“一种”，不应重复生成“一种一种……”。名称应与权利要求主题一致，通常为“产品、装置、系统、方法、工艺、设备、模块组合”等。

### 1.2 技术领域

模板句式：

`本发明/实用新型涉及一种(T2)【】，尤其涉及一种(T3)【】。`

其中 `T2` 是上位技术领域，`T3` 是更具体的技术方向、应用场景或产品类型。生成时应写成一段简洁领域定位，不展开技术效果，不写背景缺陷。

### 1.3 背景技术

模板句式：

`（T4）【】批露了一种【】，然而，这种/这些设计存在以下问题和缺点：（T5）【】。`

其中 `T4` 是现有技术或已有方案，`T5` 是现有技术问题。模板中的“批露”应改为“披露”。生成时应先客观描述已有方案，再自然引出现有技术缺陷，避免夸大或绝对化。

### 1.4 发明内容

模板包括四类内容：

1. 目的或要解决的技术问题：`T6`。
2. 整体技术方案：`T7`。
3. 可选的另一方面或补充方案：模板中 `[0005]在本发明的另一个方面中，【】。` 可作为扩展段落，不是明确编号槽位。
4. 核心结构、步骤、模块或组合及其有益效果：`T8`、`T9`。

生成时应先写“为了解决……”，再写“为实现上述目的……”，最后写“由于采用……，从而产生……”。对于方法类发明，应把“结构”改为“步骤、流程、算法、控制逻辑或系统模块”。

### 1.5 附图说明

模板句式：

`以下结合附图对本发明/实用新型做进一步详细描述。`

`附图1是本发明/实用新型(T10)【】；`

`附图2是本发明/实用新型【】；`

生成时 `T10` 应展开为附图名称和附图内容。若用户未提供附图，不得虚构“附图1、附图2”。实用新型通常需要产品结构附图，生成前应提示或在草稿中保留待补充占位。

### 1.6 具体实施方式

模板句式：

`如图（n）所示，本发明/实用新型的优选实施方式是（T11）【】。`

`如图（m）所示，本发明/实用新型的另一优选实施方式是（T12）【】。`

`T11` 和 `T12` 分别对应两个优选实施方式。生成时应把整体方案具体化为部件连接关系、步骤顺序、参数范围、模块协同方式、使用场景或替代实施例，并与前文技术方案保持一致。

### 1.7 工作原理与工作过程

模板句式：

`本发明/实用新型的工作原理与工作过程如下：（T13）【】`

`T13` 应展开为方案如何运行、各部件或步骤如何协同、输入输出如何变化、技术效果如何实现。方法类方案可写“执行流程”“控制逻辑”“算法流程”；结构类方案可写“装配关系、动作路径、工作状态转换”。

## 2. T1-T13 槽位含义

| 槽位 | 含义 | 模板位置 | 生成作用 |
|---|---|---|---|
| T1 | 专利名称 | 标题、发明内容段落 | 确定全文主题、技术方案称谓和权利要求主题 |
| T2 | 上位技术领域 | 技术领域 | 给出较宽的技术分类 |
| T3 | 具体技术方向、应用场景或产品类型 | 技术领域 | 收窄到具体应用或产品形态 |
| T4 | 背景技术或已有方案 | 背景技术 | 描述现有技术基础，建立改进对象 |
| T5 | 现有技术存在的问题和缺点 | 背景技术 | 引出发明目的和技术问题 |
| T6 | 发明目的或要解决的技术问题 | 发明内容 | 回应背景缺陷，形成问题导向 |
| T7 | 整体技术方案 | 发明内容 | 概括本申请的总体实现方式 |
| T8 | 核心结构、核心步骤、关键模块或关键组合 | 发明内容 | 说明创造性贡献的核心组成 |
| T9 | 有益效果 | 发明内容 | 说明方案带来的技术效果 |
| T10 | 附图名称和附图内容 | 附图说明 | 生成附图说明条目，需以真实附图为准 |
| T11 | 优选实施方式一 | 具体实施方式 | 将整体方案落到第一个可实施例 |
| T12 | 优选实施方式二 | 具体实施方式 | 提供变形例、替代例或另一应用例 |
| T13 | 工作原理和工作过程 | 具体实施方式末尾 | 说明方案运行机制和工作流程 |

## 3. 用户输入字段与生成输出字段

| 槽位 | 用户最好提供什么 | skill 应如何展开 |
|---|---|---|
| T1 | 拟申请的专利名称或核心主题 | 统一为规范标题，避免重复“一种”，并贯穿全文 |
| T2 | 所属上位技术领域 | 写成技术领域首句的上位分类 |
| T3 | 具体应用场景、产品类型或技术方向 | 写成“尤其涉及……”的具体领域定位 |
| T4 | 已有产品、已有方法、传统方案或公开技术 | 客观改写为背景技术，不评价本申请 |
| T5 | 现有方案痛点、缺陷、限制或不足 | 整理为若干技术问题，避免商业化空话 |
| T6 | 希望解决的主要问题或发明目的 | 写成“为了解决……”句式，与 T5 对应 |
| T7 | 总体方案、总体流程、系统组成或装置构成 | 生成发明内容中的整体技术方案段落 |
| T8 | 最核心的结构、步骤、模块、算法或组合关系 | 根据主题类型改写为结构、步骤、流程、控制逻辑或模块协同 |
| T9 | 预期技术效果、性能改善或使用效果 | 写成可由 T8 推导出的有益效果 |
| T10 | 附图编号、图名、图中展示内容 | 仅在真实存在附图时生成附图说明；缺失时保留待补充提示 |
| T11 | 第一个优选实施例的细节 | 展开为具体实施方式一，补足连接关系、流程或参数 |
| T12 | 第二个优选实施例、变形例或替代例 | 展开为具体实施方式二，与 T11 区分但不脱离总方案 |
| T13 | 工作原理、操作步骤、运行过程或控制过程 | 写成连贯工作过程，说明技术效果如何产生 |

用户输入字段偏向“事实与约束”：`T1-T13` 都可以由用户提供，尤其应优先提供 `T1`、`T3`、`T5`、`T7`、`T8`、`T9`、`T10`、`T11`、`T13`。

生成输出字段偏向“专利文本展开”：skill 应把用户事实转换为说明书标题、各章节段落、附图说明、实施例和工作过程；对缺失信息可以合理概括，但不得虚构附图、实验数据、未给出的部件或无法从方案推出的效果。

## 4. 模板风险识别

1. 错别字风险：模板中的“批露”应修正为“披露”。
2. 专利类型风险：模板反复使用“本发明/实用新型”，生成时必须根据专利类型统一为“本发明”或“本实用新型”。
3. 附图虚构风险：没有附图时不得生成“附图1、附图2”等虚构图名或图示内容。
4. 实用新型附图风险：实用新型通常需要产品结构附图；若用户未提供，应提示补充结构图、部件标号图或装配关系图。
5. 主题类型风险：方法类发明不能机械套用“结构”，应改为“步骤、流程、算法、控制逻辑或系统模块”。
6. 空槽位风险：模板中 `[0005]在本发明的另一个方面中，【】。` 没有 T 编号，生成时应作为可选补充方案，不应保留空括号。
7. 标题重复风险：`一种(T1)` 与用户标题可能重复“一种”，生成时需去重。
8. 段落编号风险：模板段落编号较少，生成完整说明书时应根据实际段落顺序连续编号。
9. 标点风险：模板中“……。。”应清理为规范省略号或删除。

## 5. 文件末尾输出

### 5.1 T1-T13 槽位映射表

| 槽位 | 用户输入字段 | 生成输出字段 |
|---|---|---|
| T1 | 专利名称、主题名称 | 标题、发明内容中的方案名称、全文统一称谓 |
| T2 | 上位技术领域 | 技术领域第一层分类 |
| T3 | 具体技术方向、应用场景或产品类型 | 技术领域具体化描述 |
| T4 | 背景技术或已有方案 | 背景技术中的现有方案描述 |
| T5 | 现有技术问题和缺点 | 背景技术末尾的问题引出 |
| T6 | 发明目的或待解决问题 | 发明内容的目的段 |
| T7 | 整体技术方案 | 发明内容的总体方案段 |
| T8 | 核心结构、步骤、模块或组合 | 发明内容中的核心贡献说明 |
| T9 | 有益效果 | 发明内容的效果段 |
| T10 | 附图名称、编号、内容 | 附图说明条目 |
| T11 | 优选实施方式一事实 | 具体实施方式第一实施例 |
| T12 | 优选实施方式二事实 | 具体实施方式第二实施例或变形例 |
| T13 | 工作原理和工作过程事实 | 工作原理、运行流程、操作过程或控制逻辑 |

### 5.2 说明书生成顺序

1. 判断并统一专利类型：发明使用“本发明”，实用新型使用“本实用新型”。
2. 生成专利名称：根据 `T1` 形成规范标题，处理“一种”重复问题。
3. 生成技术领域：用 `T2` 和 `T3` 写一段领域定位。
4. 生成背景技术：用 `T4` 描述已有方案，用 `T5` 提炼问题和缺点，并修正“披露”用词。
5. 生成发明内容目的段：用 `T6` 回应背景技术问题。
6. 生成整体技术方案：用 `T7` 概括方案框架。
7. 生成核心贡献与有益效果：用 `T8` 和 `T9` 写核心结构、步骤、模块或控制逻辑及技术效果。
8. 判断是否有附图：有真实附图时用 `T10` 生成附图说明；无附图时不虚构图号。
9. 生成具体实施方式：用 `T11` 和 `T12` 展开优选实施例、变形例或替代实施例。
10. 生成工作原理与工作过程：用 `T13` 说明运行机制、操作过程、流程或控制逻辑。
11. 清理模板残留：删除空括号、未填槽位、重复标点和不适用的“结构”表述。

### 5.3 应写入 SKILL.md 的说明书规则

1. 说明书生成必须包含：专利名称、技术领域、背景技术、发明内容、附图说明、具体实施方式、工作原理与工作过程。
2. 使用 `T1-T13` 作为说明书骨架字段，但生成文本不能保留 `T` 占位符。
3. 专利类型必须统一称谓，不得输出“本发明/实用新型”这种二选一模板文本。
4. 背景技术中应使用“披露”，不得使用错别字“批露”。
5. 没有真实附图时，不得虚构附图编号、图名、图中内容或“如图所示”。
6. 实用新型草稿应优先要求或保留产品结构附图信息。
7. 方法类发明应把“结构”改写为“步骤、流程、算法、控制逻辑或系统模块”。
8. `T8` 与 `T9` 必须有因果关系：有益效果应由核心结构、步骤或模块组合推导而来。
9. `T11`、`T12` 应是对 `T7`、`T8` 的具体化，不得引入与整体方案冲突的新主题。
10. `T13` 应说明工作原理和工作过程，不能只重复有益效果。

### 5.4 应写入 templates/patent_draft_output.md 的说明书模板要点

1. 标题模板：`一种{T1}`，但若 `{T1}` 已含“一种”，应自动去重。
2. 技术领域模板：`本{专利类型称谓}涉及{T2}，尤其涉及{T3}。`
3. 背景技术模板：先写 `{T4} 披露/公开/采用了……`，再写 `然而，现有方案存在{T5}`。
4. 发明内容目的模板：`为了解决上述问题，本{专利类型称谓}提供了一种{T1}，其目的在于{T6}。`
5. 技术方案模板：`为实现上述目的，本{专利类型称谓}的{T1}包括/采用/通过{T7}。`
6. 核心贡献与效果模板：`由于本{专利类型称谓}采用{T8}，从而能够{T9}。`
7. 附图说明模板：仅当存在真实附图时输出 `附图{编号}是本{专利类型称谓}{图名或图示内容}`。
8. 具体实施方式模板：用 `{T11}`、`{T12}` 分别生成优选实施方式和另一优选实施方式。
9. 工作原理模板：`本{专利类型称谓}的工作原理与工作过程如下：{T13}`。
10. 方法类替换规则：将“结构、部件、连接关系”替换或扩展为“步骤、流程、算法、控制逻辑、模块协同”。
