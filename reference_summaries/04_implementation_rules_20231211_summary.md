# 《中华人民共和国专利法实施细则（2023年修订）》初稿生成规则摘要

来源文件：`references/中华人民共和国专利法实施细则_20231211.docx`

处理目标：提取与发明、实用新型专利初稿生成直接相关的细化规则，供后续写入技能规则、法律依据、自检清单和 T1-T13 模板时使用。

## 1. 文件提交基本规则

依据：第2条、第3条、第4条、第43条、第44条、第50条。

- 各种手续原则上应以书面形式或者国务院专利行政部门规定的其他形式办理。
- 以电子数据交换等方式形成、能够有形表现所载内容、并可随时调取查用的数据电文，属于“电子形式”，视为书面形式。
- 以电子形式提交文件的，递交日为文件进入国务院专利行政部门指定电子系统的日期。
- 提交的各种文件应使用中文。
- 国家已有统一规定的科技术语，应采用规范词。
- 外国人名、地名、科技术语没有统一中文译文的，应注明原文。
- 外文证件或证明文件在必要时可能被要求补交中文译文；期满未补交的，视为未提交该证件或证明文件。
- 发明或实用新型申请的基本申请文件包括请求书、说明书、权利要求书；实用新型说明书必须包括附图。
- 未使用中文、缺少必要申请文件、实用新型无附图、请求书缺少申请人姓名/名称或地址、申请类别不明确等，可能导致申请不予受理。
- 初步审查会检查申请文件是否符合电子/书面形式、中文、术语规范等基本要求。

## 2. 请求书信息

依据：第17条、第19条、第34条、第35条、第37条、第43条、第44条、第146条、第147条。

请求书应写明：

- 发明、实用新型或者外观设计的名称。
- 申请人信息：
  - 中国单位或个人：名称/姓名、地址、邮政编码、统一社会信用代码或者身份证件号码。
  - 外国人、外国企业或者外国其他组织：姓名/名称、国籍或者注册的国家或地区。
- 发明人或者设计人的姓名。
- 委托专利代理机构的，写明受托机构名称、机构代码，以及指定专利代理师姓名、资格证号码、联系电话。
- 要求优先权的，写明在先申请的申请日、申请号、原受理机构名称。
- 申请人或者专利代理机构的签字或者盖章。
- 申请文件清单。
- 附加文件清单。
- 其他需要写明的事项。

生成规则：

- 初稿生成请求书信息时，应保证专利名称与说明书名称一致。
- 多申请人且未委托代理机构时，除请求书另有声明外，以请求书指明的第一申请人为代表人。
- 要求本国优先权时，请求书中写明在先申请申请日和申请号的，视为提交在先申请文件副本。
- 优先权信息漏写或错写在先申请申请日、申请号、原受理机构名称中的一项或两项，可能进入补正；期满未补正的，视为未要求优先权。
- 除首次提交专利申请文件外，后续提交文件应标明申请号或专利号、发明创造名称、申请人或专利权人姓名/名称。

## 3. 说明书规则

依据：第20条、第27条、第29条、第43条、第44条、第50条。

说明书应写明发明或者实用新型名称，且名称应与请求书中的名称一致。发明或实用新型说明书通常应按照下列方式和顺序撰写，并在各部分前写明标题：

- 技术领域：写明要求保护的技术方案所属的技术领域。
- 背景技术：写明有助于理解、检索、审查的背景技术；可行时引证反映这些背景技术的文件。
- 发明内容：写明要解决的技术问题、解决该技术问题采用的技术方案，并对照现有技术写明有益效果。
- 附图说明：有附图的，对各幅附图作简略说明。
- 具体实施方式：详细写明申请人认为实现发明或实用新型的优选方式；必要时举例说明；有附图的，应对照附图。

撰写禁忌和质量要求：

- 用词应规范，语句应清楚。
- 不得使用“如权利要求……所述的……”一类引用语。
- 不得使用商业性宣传用语。
- 除非发明或实用新型性质决定其他方式或顺序更节约篇幅且能使他人准确理解，否则应按上述结构撰写。
- 实用新型说明书应有表示要求保护产品的形状、构造或者其结合的附图。
- 涉及生物材料样品保藏、遗传资源等特殊主题时，请求书和说明书还可能需要补充特定信息；常规初稿生成可作为专项扩展规则处理。

## 4. 附图规则

依据：第20条、第21条、第26条、第43条、第44条、第46条。

- 发明或实用新型有几幅附图的，应按“图1、图2……”顺序编号排列。
- 说明书文字部分未提及的附图标记不得出现在附图中。
- 附图中未出现的附图标记不得在说明书文字部分提及。
- 申请文件中表示同一组成部分的附图标记应一致。
- 附图中除必需词语外，不应含有其他注释。
- 说明书中写有附图说明但无附图或缺少部分附图的，申请人可能被要求补交附图或取消相应附图说明。
- 实用新型申请中，附图不是可选项；无附图会影响受理。

初稿生成约束：

- 生成说明书文字、附图说明和权利要求时，应维护同一套附图标记表。
- 不应生成只在图中出现、正文未解释的编号。
- 不应在正文提及图中没有的编号。
- 同一部件不能在不同位置使用不同附图标记。
- 附图中的文字应尽量限于必要的名称、方向或流程标识；不应写宣传语、解释性长句或效果评价。

## 5. 权利要求书规则

依据：第22条、第23条、第24条、第25条、第43条、第50条、第73条。

基础规则：

- 权利要求书应记载发明或者实用新型的技术特征。
- 有多项权利要求的，应使用阿拉伯数字顺序编号。
- 权利要求中的科技术语应与说明书中使用的科技术语一致。
- 权利要求书可以有化学式或者数学式，但不得有插图。
- 除绝对必要外，不得使用“如说明书……部分所述”或者“如图……所示”。
- 权利要求中的技术特征可以引用说明书附图中相应标记；附图标记应放在相应技术特征之后并置于括号内。
- 附图标记只用于便于理解权利要求，不得解释为对权利要求的限制。

独立权利要求：

- 权利要求书应有独立权利要求，也可以有从属权利要求。
- 独立权利要求应从整体上反映发明或者实用新型的技术方案。
- 独立权利要求应记载解决技术问题的必要技术特征。
- 通常包括前序部分和特征部分：
  - 前序部分写明主题名称以及与最接近现有技术共有的必要技术特征。
  - 特征部分用“其特征是……”或类似用语，写明区别于最接近现有技术的技术特征。
- 发明或者实用新型的性质不适合前序/特征写法的，独立权利要求可以采用其他方式。
- 一项发明或者实用新型应只有一个独立权利要求，并写在其从属权利要求之前。

从属权利要求：

- 从属权利要求应使用附加技术特征，对引用的权利要求进一步限定。
- 从属权利要求应包括引用部分和限定部分。
- 引用部分应写明引用的权利要求编号及其主题名称。
- 限定部分应写明附加技术特征。
- 从属权利要求只能引用在前的权利要求。
- 多项从属权利要求只能以择一方式引用在前权利要求。
- 多项从属权利要求不得作为另一项多项从属权利要求的基础。

## 6. 摘要规则

依据：第26条。

说明书摘要应写明发明或者实用新型专利申请公开内容的概要，至少包括：

- 发明或者实用新型名称。
- 所属技术领域。
- 所要解决的技术问题。
- 解决该问题的技术方案要点。
- 主要用途。

附加规则：

- 摘要可以包含最能说明发明的化学式。
- 有附图的专利申请，应在请求书中指定一幅最能说明该发明或者实用新型技术特征的说明书附图作为摘要附图。
- 摘要不得使用商业性宣传用语。

## 7. 实用新型特别规则

依据：第20条、第43条、第44条、第50条；并由第50条引用的专利法第2条第3款形成初审合规检查。

- 实用新型申请文件必须包括附图。
- 国务院专利行政部门收到实用新型申请的请求书、说明书（必须包括附图）和权利要求书后，才进入明确申请日、给予申请号并通知申请人的处理路径。
- 实用新型缺少请求书、说明书、权利要求书，或者说明书无附图的，属于不予受理情形。
- 实用新型说明书中的附图应表示要求保护产品的形状、构造或者其结合。
- 初步审查会检查实用新型是否符合本细则第19条至第26条，以及是否明显不符合专利法第2条第3款等规定。

初稿生成规则：

- 实用新型应围绕产品的形状、构造或者其结合来组织核心保护对象。
- 方法、流程、算法、用途、控制策略本身不应作为实用新型的核心保护对象。
- 如果用户输入主要是方法、流程、算法或控制策略，应提示其更适合发明专利；若仍生成实用新型草稿，应将保护重心转化为具有具体结构关系的产品、装置、系统部件或机构组合。
- 实用新型的说明书、权利要求书和附图说明应始终能对应到可见或可描述的结构特征。
- 仅描述功能效果而缺少结构、连接关系、空间关系或部件组合的实用新型初稿，应判为风险稿。

## 8. 文件末尾输出

### 应写入 SKILL.md 的规则

- 生成专利初稿时，所有申请文件默认使用中文；术语采用规范科技术语；外国人名、地名、无统一译名的科技术语应补充原文。
- 请求书、说明书、权利要求书、摘要、附图说明之间必须保持名称、术语、附图标记一致。
- 说明书按“技术领域、背景技术、发明内容、附图说明、具体实施方式”组织，除非技术性质决定其他顺序更清楚。
- 说明书不得出现“如权利要求……所述的……”一类引用语，不得出现商业性宣传用语。
- 权利要求书必须以技术特征撰写；多项权利要求用阿拉伯数字编号；不得插图；术语与说明书一致。
- 独立权利要求应覆盖整体技术方案并写明必要技术特征；从属权利要求只能引用在前权利要求，多项从属权利要求只能择一引用且不得作为另一多项从属权利要求基础。
- 附图标记应在说明书、附图说明、权利要求中双向一致；同一组成部分使用同一标记。
- 摘要写明名称、技术领域、技术问题、技术方案要点和主要用途，不得使用商业性宣传用语；有附图时指定摘要附图。
- 实用新型必须有附图，且保护对象应落在产品形状、构造或者其结合上；方法、流程、算法、用途、控制策略不作为实用新型核心保护对象。

### 与 T1-T13 模板需要合并的地方

未读取 T1-T13 模板文件；以下按专利初稿生成流程给出应合并的规则位置，需在合并时映射到现有模板编号和字段名。

- 请求书信息模板：合并第19条字段，增加名称、申请人、发明人/设计人、代理机构/代理师、优先权、申请文件清单、附加文件清单的必填/可选规则。
- 标题/名称模板：合并“请求书名称、说明书名称、摘要名称、权利要求主题名称一致”的校验。
- 技术领域模板：合并“写明要求保护技术方案所属技术领域”的要求。
- 背景技术模板：合并“有助于理解、检索、审查；可引证背景文件”的要求，避免宣传性背景。
- 发明内容模板：合并“技术问题、技术方案、有益效果”三段式要求。
- 附图说明模板：合并图号顺序、附图标记双向一致、同一部件标记一致、附图注释限制。
- 具体实施方式模板：合并“详细写明优选实施方式、必要时举例、有附图时对照附图”的要求。
- 权利要求模板：合并技术特征、阿拉伯数字编号、术语一致、不得插图、引用语限制、括号内附图标记、独立/从属权利要求规则。
- 独立权利要求模板：合并整体技术方案、必要技术特征、前序部分/特征部分、“其特征是……”等规则。
- 从属权利要求模板：合并引用部分、限定部分、只能引用在前权利要求、多项从属权利要求择一引用且不得作为另一多项从属权利要求基础的规则。

- 实用新型模板：合并必须有附图、产品形状/构造/结合导向、排除方法/流程/算法/用途/控制策略作为核心保护对象的规则。
- 终稿自检模板：合并中文、术语、名称一致、附图标记一致、商业性宣传用语、禁止引用语、实用新型附图和保护对象、摘要附图等检查项。
