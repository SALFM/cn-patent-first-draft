# Phase 1：发明挖掘与技术闭环构建

## 目标

从用户材料中挖掘核心创新点，建立“现有技术缺陷 -> 技术问题 -> 技术手段 -> 技术效果”的闭环。

## 输入

- Phase 0 的结构化输入摘要。
- 用户提供的现有技术、缺陷、技术方案、效果和实施例。

## 输出

- 发明画像。
- 候选创新点列表。
- 区别技术特征与技术效果因果链。
- 现有技术压力测试假设或用户提供现有技术的 element-by-element 对比。

## 禁止事项

- 不得把压力测试假设写成真实检索结论。
- 不得把商业效果、管理目标或主观体验直接写成技术效果。
- 不得把未提供的实验数据写入效果论证。

## 质量门

- 每个主要技术问题至少对应一个技术手段和一个技术效果。
- 核心特征与效果之间有明确因果关系。
- 无法确认的区别特征标记为 `[待补充]` 或“建议确认”。

## 失败时如何处理

如因现有技术缺失无法确定区别特征，生成“压力测试假设”，并将现有技术缺失路由到 Prior Art Pressure Test。
