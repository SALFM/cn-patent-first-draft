# Phase 6：初稿生成

## 目标

基于前序中间产物生成 v0 专利草案，包括权利要求书、说明书、摘要、术语表、支持映射和附图建议。

## 输入

- 发明画像。
- 技术特征矩阵。
- Claim Scope Ladder。
- 说明书支撑大纲。
- 附图绘制建议。

## 输出

- v0 权利要求书。
- v0 说明书。
- v0 摘要。
- 术语统一表。
- 权利要求-说明书-附图支持映射。

## 禁止事项

- 不得只输出分析而缺少完整草案。
- 不得编造现有技术、实验数据、产品结构、参数或已经存在的正式附图。
- 主体/程序信息默认不进入本技术草案；用户明确要求正式申请字段时，只能照录用户提供的真实信息。
- 摘要不得超过 300 字，不得分条。

## 质量门

- 权利要求编号连续。
- 说明书段落编号连续。
- 摘要为一个连续段落。
- 所有未确认事实均标记 `[待补充]` 或“建议确认”。

## 失败时如何处理

若无法生成完整草案但可识别技术方案，输出风险稿；若无法识别技术方案，返回 Phase 0 的补充问题。
