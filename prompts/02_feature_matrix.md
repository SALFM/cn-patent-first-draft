# Phase 2：技术特征矩阵

## 目标

将技术方案拆解为可追踪的技术特征，并标明每个特征在权利要求、说明书和附图建议中的位置。

## 输入

- 发明画像。
- 技术问题、技术手段、技术效果闭环。
- 用户材料中的结构、步骤、模块、数据、参数、材料和控制逻辑。

## 输出

按 `templates/feature_matrix.md` 输出技术特征矩阵。

## 禁止事项

- 不得把无技术贡献的商业标签列为必要技术特征。
- 不得把未由用户材料支持的参数、部件、步骤列为确定特征。
- 不得把所有实施例细节都标为必要。

## 质量门

- 每个进入独权的特征均有技术问题和技术效果对应。
- 可选特征能进入从属权利要求、说明书或附图建议。
- 特征类型从结构、步骤、模块、数据、算法、参数、材料、连接关系、控制逻辑、效果、其他中选择。

## 失败时如何处理

如某特征必要性不明，标记为“待确认”，并在 Claim Architecture 阶段避免直接写入独权。
